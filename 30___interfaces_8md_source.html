4 \section interfaces_idirectplugin IDirectPlugin Interface
5 
6 The IDirectPlugin interface is the main interface for plugins. All plugins must implement this interface. Otherwise they will not be recognized by the B2S.Server or other applications hosting the plugin.
7 
8 The interface implementation looks as follows (if the comment are removed):
9 
10 ~~~~~~~~~~~~~{.cs}
11  public interface IDirectPlugin
12  {
13  string Name { get; }
14 
15  void PluginInit(string TableFilename, string RomName = "");
16 
17  void PluginFinish();
18 
19  void DataReceive(char TableElementTypeChar, int Number, int Value);
20  }
21 ~~~~~~~~~~~~~

25 \section interfaces_idirectpluginpinmame IDirectPluginPinMame Interface

47 \section interfaces_idirectpluginfrontend IDirectPluginFrontEnd Interface
48 
49 If a plugin has some frontend (e.g. configuration window) it is a good choice to implement the IDirectPluginFrontEnd interface. This interface will allow the host application to open the frontend when necessary.
50 
51 The optional Owner parameter (make sure your implementation does also accept calls with null for the parameter) is used to pass a reference to the owner window for the frontend. This allows to ensure that your frontend windows are centered on the call window and/or that the frontend windows are put on the same screen as the calling appilcation.
52 
53 ~~~~~~~~~~~~~{.cs}
54  public interface IDirectPluginFrontend
55  {
56  void PluginShowFrontend(Form Owner = null);
57  }
58 ~~~~~~~~~~~~~
